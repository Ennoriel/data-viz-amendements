Le projet de loi climat est-il écologique ?

Pour cet article, je souhaite m'intéresser à la page de consultation public du projet de loi tel qu'il est consultable ici  (https://www.assemblee-nationale.fr/dyn/docs/PRJLANR5L15BTC3995.raw). Cette page est utilisée pour visualiser les articles du projet au sein du dossier législatif du site de l'assemblée nationale (https://www.assemblee-nationale.fr/dyn/15/textes/l15b3995_texte-adopte-commission).

Le but est de regarder ici l'éco-conception de cette page unique. Ce travail est volontairement réducteur vis à vis d'un travail plus global qui pourrait être fait d'audit de l'éco-conception du site de l'assemblée nationale.

La page est statique. Elle est rendue de manière statique, c'est à dire qu'il n'y a pas besoin de télécharger de scripts pour visualiser cette page dans un navigateur. C'est un très bon point. En particulier, par exemple, les développeurs des framework récent ne prennent pas toujours en compte cet aspect.

La page HTML ne télécharge pas de fichier de style supplémentaire, celui-ci est encapsulé dans le header et dans le style in-line. C'est un choix qui n'est pas toujours négatif. En particulier, ici, on suppose que le style n'est pas dupliqué mais bien injecté, ce qui évide la redondance du code.

Néanmoins, sur le style commun :
	- des styles sont importés et non utilisés (plus de la moitié)
	- des balises css sont vides (un sélecteur sans style)

Pour le style in-line, celui-ci est extrêmement redondant (alignement, taille des marges pour les tableaux, etc.) et pourraient être passé en style global.

Hormis le style, trois éléments supplémentaires sont à noter :
	- la page n'a pas été mimifié (il reste les espaces et tabulations inutiles) => gain ed 17 ko / 815 ko
	- les espaces insécables sont insérés dans des balises span (<span>&#xa0;</span>) et n'ont pas d'utilité au niveau du style => gain de 4 ko / 815 ko
	- les numéros d'alniéa sont représentés à partir d'image en base 64 en dur dans le code source. Pour représenter un élément qui peut être définis en css pur, l'utilisation d'une image est excessive. Il n'y a par ailleurs pas de mutualisation. La suppression de ces images représente un gain de près de 400 ko / 815 ko (contre moins d'1 ko si l'on souhaite le réaliser en css)
